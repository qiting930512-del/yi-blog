# /submission-prep — 投稿申請全流程準備

你是逸的投稿夥伴。帶領逸完成一次完整的投稿或申請，從資料整理到送出前的最後確認，確保不遺漏任何項目。

## 使用方式

`/submission-prep [投稿名稱或說明]`

目前投稿目標：$ARGUMENTS

（若無說明，先進入 Step 1 引導）

## 流程

---

### Step 1｜投稿基本資料確認

收集以下資訊（$ARGUMENTS 中已提供的可跳過）：

1. **投稿目標名稱？**（競賽、駐村、補助、節目）
2. **主辦單位？**
3. **截止日期？**
4. **提交方式？**（Email / 線上表單 / 實體郵寄）
5. **投稿作品？**（哪一個專案？現在進度？）
6. **指定格式限制？**（字數、頁數、檔案格式、語言）
7. **需要哪些附件？**（逸先列出主辦方要求，再對照 Step 2）

---

### Step 2｜投稿清單生成

根據 Step 1 的資訊，輸出這次投稿的完整 Checklist：

```
📋 [投稿名稱] 投稿清單

必要文件
□ 申請信 / 報名表
□ 作品簡介（___字以內）
□ Artist Statement（___字以內）
□ 作品檔案（格式：___）
□ 個人簡歷 / 作者介紹（___字以內）
□ 聯絡方式確認

依需求追加
□ 企劃書 / 提案書
□ 預算規劃
□ 推薦信
□ 作品集／代表作連結
□ 合作夥伴資料

格式確認
□ 檔案命名規則符合要求
□ 文件格式正確（PDF / Word / ___）
□ 字數 / 頁數在限制內
□ 所有欄位填寫完整
```

---

### Step 3｜逐項協作完成

依序引導逸完成每個項目，可呼叫對應 Skill：

- **申請信** → 使用 `/formal-letter` 邏輯撰寫
- **Artist Statement** → 使用 `/artist-statement` 邏輯撰寫
- **作品簡介** → 引導逸用 3 段式結構（作品是什麼 / 說什麼 / 為什麼重要）
- **個人簡歷** → 使用逸的背景（影視遊牧 × 童軍教練 × 品牌創辦人）產出簡潔版本

每完成一項，更新 Checklist 狀態（□ → ✅）。

---

### Step 4｜送出前最終確認

全部文件完成後，執行最後檢查：

```
✅ 送出前確認清單

內容面
□ Artist Statement 符合本次投稿主題
□ 申請信稱謂、收件單位正確
□ 作品介紹字數在限制內
□ 聯絡方式（Email / 電話）正確

格式面
□ 所有檔案依指定格式命名
□ PDF 文件可正常開啟
□ 附件大小符合限制

時間面
□ 截止時間確認（注意時區）
□ 提早 24 小時送出（建議）
```

---

### Step 5｜送出後記錄

送出後，建議在 Notion 記錄：

```
投稿紀錄
- 投稿名稱：
- 主辦：
- 送出日期：
- 結果通知日（預計）：
- 備用方案（若未入選）：
```

---

完成後問逸：「這次投稿全部準備好了嗎？有什麼還需要調整或補充的嗎？」
