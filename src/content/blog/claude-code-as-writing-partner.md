三個月前，我開始用 Claude Code 管理我的工作。

不是讓它幫我寫文章（雖然也有），而是讓它變成我整個工作系統的「神經中樞」。

這是我的實驗報告。

---

## 背景：我的問題

我同時在做三件事：

1. 影視製作（拍片、剪接、客戶提案）
2. 品牌經營（破繭的 IG、逸的個人品牌）
3. 劇本創作（長期在跑的世界觀開發）

這三件事都重要，但它們的節奏不一樣。

拍片是爆發性的。劇本是長期的。品牌經營需要每天有一點點。

我一直找不到一個系統，可以同時管理這三個節奏。

---

## 轉捩點

我試過很多工具：Notion、Obsidian、Apple Notes、紙本子。

都用過。都沒有堅持下來。

不是工具不好，是「切換工具」這件事本身太貴了。

Claude Code 的出現讓我重新思考：如果工具可以**找我**，而不是我去找工具呢？

---

## 我現在的工作流

### 1. Daily Log

每天晚上，我用 `/daily-log` 跟 Claude 說今天做了什麼。

Claude 會整理成一份結構化的紀錄，存進 Notion。

重點不是紀錄「我做了什麼」，而是通過說出來，**發現今天其實發生了什麼**。

### 2. 晨間日報

每天早上的 `/morning`，會彙整 Gmail、Google Calendar、Notion 任務。

5 分鐘之內，我知道今天什麼最重要。

### 3. 部落格發佈

這篇文章就是一個例子。

我在 Notion 裡有一個「寫作草稿」資料庫。當一篇草稿準備好，我告訴 Claude 發佈，它幫我轉成 Markdown、push 到 GitHub，Vercel 自動部署。

---

## 最意外的收穫

我以為 Claude Code 會讓我更「有效率」。

實際上，它讓我更**清醒**。

因為每次對話我都要說清楚我想做什麼，這個過程本身讓我不得不思考：

*這件事現在真的重要嗎？*

這比任何任務管理 app 都更有效。

---

## 還在實驗的部分

- 自動從 Notion 草稿同步到部落格（目前還是手動確認）
- 影視拍攝前的 pre-production checklist 自動化
- 劇本寫作助手（試驗中，還沒穩定）

---

工具是為了讓你更像自己，不是讓你更像工具。

這三個月，我好像慢慢在找回一些東西。
